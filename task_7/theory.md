# Сборка (build)

* Сборка - это процесс получения итогового файла проекта, готового для использования
  * исполняемый файл
  * библиотека, которую можно подключить

* Классический вариант исполняемого файла - `.exe`
* Классический вариант библиотеки - `.dll`

* В Java такой вариант исполняемого файла или библиотеки мы не получаем в таких вариантах
* Поскольку Java исполняется внутри виртуальной машины, то и исполняемый файл и библиотека
  * `.jar` - Java Archive, по сути сжатая zip-папка с необходимыми файлами
  * Если это исполняемый файл, значит там есть main-метод
  * Если это библиотека, то его там нет

* Сборка в Java подразумевает следующее:
  * Подключение библиотек 
  * Компиляция (`.java` -> `.class`)
  * Упаковка в архив `.jar`

## Maven

* Наиболее распространенный инструмент сборки в Java - `Maven`
* `Maven` это отдельное приложение (с IntelliJ IDEA идет вместе)
* Оно умеет работать с удаленными репозиториями и собирать проект
* Важное понятие - Maven подразумевает наличие `координат` для любого проекта
  * `groupId` - идентификатор компании, как правило `reverse-domain-naming`, ex. `ru.itis`, `org.postgres`
  * `artifactId` - идентификатор конкретного проекта
  * `version` - версия проекта

### Проект на Maven

* `src` - файлы исходного кода и ресурсы приложения, эта папка проходит фазу `компиляции`
  * `main` - основные файлы
    * `java` - исходный код на Java
    * `resources` - файлы ресурсов
  * `test` - файлы модульных тестов
* `target` - папка с результатами компиляций и сборок
* `pom.xml` - файл проекта

* `.m2` - папка локального репозитория, хранит скачанные архивы из центрального репозитория
