# СУБД

* Системы управления базами данных
* Глобально нужны для того, чтобы эффективно хранить и работать с данными
* В отличие от обычных текстовых файлов СУБД содержат логику обработки данных, 
  * специальные приложения для администрирования, а также эффективные средства для взаимодействия с данными

* Виды СУБД
  * Реляционные - набор таблиц, состоящих из строк и столбцов (аналог Excel)
  * Нереляционные (MongoDB - JSON, Redis - ключ-значение) - подразумевает хранение данных в неструктурированном формате, например в JSON

```json
{
  "data" : {
    "array": ["marsel", "sidikov"],
    "isTrue": true,
    "real": 7.5
  },
  
  "id": 1
}
```
## Реляционные SQL-базы данных

* Состоят из строк и столбцов
* Столбцы имеют тип и название, а также дополнительные ограничения, контролируемые самой СУБД
* Управляется каким-либо диалектом языка SQL

```
ACCOUNT

id|first_name|last_name|age

3|Marsel|Sidikov|32

Ограничения - имя и фамилия не могут быть пустыми, возраст строго от 0 до 180
```

* Таких СУБД очень много разных вариантов
* Например: MySQL, Oracle, PostgreSQL, SQLite, H2, HSQLDB и т.д.
* В каждой из СУБД используется SQL, но определенный его диалект

* Путь к базе данных:
* Сервер - база данных - схема PUBLIC - непосредственно таблицы

## SQL-запросы

* Создание таблицы

```sql
create table account (
                         id serial unique not null,
                         first_name char(20) not null,
                         last_name char(20) not null,
                         age integer check(age > -1 and age < 181),
                         is_active boolean
);
```

* Добавление данных

```sql
insert into account (first_name, last_name, age) values ('Marsel', 'Sidikov', 32);
```

* Получение данных

```sql
select * from account;
```

* Удаление таблицы

```sql
drop table account;
```
